# Changelog

Release notes are generated automatically from each `v*` tag and attached to
the matching GitHub Release.

The release workflow also writes the same notes into a `CHANGELOG.md` asset so
the history stays bundled with the tag release.

## Current Policy

- tags drive releases
- GitHub Releases are created from tags
- release notes are auto-generated
- changelog content comes from the same generated notes

For the release process itself, see [Release Guide](release.md).
